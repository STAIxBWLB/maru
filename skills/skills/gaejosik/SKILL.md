# 개조식 글쓰기 (명사형 종결)

한국 공식 보고서와 문건에서 사용하는 개조식 스타일. 핵심: 문장 종결어를 명사형으로 마무리.

## 개조식이란

한국의 공적 문서(정부 보고서, 기획서, 사업계획서 등)에서 표준적으로 사용하는 글쓰기 양식. "~다", "~함", "~임" 등의 서술형 종결 대신, 명사 또는 명사구로 문장 종결. 간결하고 객관적인 인상을 주며, 정보 밀도 향상.

개조식은 짧은 문장만 의미하지 않음. 긴 문장도 가능하며, 핵심은 종결어가 명사형이라는 점.

## 금지 종결어

다음 세 가지 유형의 종결어는 개조식에서 모두 금지:

1. **서술형 "~다" 종결** — "~이다", "~한다", "~된다", "~있다", "~않는다", "~높인다" 등
2. **축약 서술형 "~함/~임/~됨"** — "~필요함", "~상태임", "~확인됨" 등
3. **경어체** — "~합니다", "~됩니다", "~있습니다" 등

이 세 유형 모두 명사/명사구 종결로 변환해야 함.

## 종결어 변환 규칙

### "~다" 서술형 → 명사형

| 서술형 (쓰지 않음) | 개조식 (올바른 형태) |

| ~구현한다 | ~구현 |

| ~존재한다 | ~존재 |

| ~예정이다 | ~예정 |

| ~밀도를 높인다 | ~밀도 향상 |

## 문장 길이와 구조

개조식은 짧은 문장만 의미하지 않음. 복잡한 내용도 명사형 종결로 표현 가능.

**짧은 개조식:**
- 2025년 8월 착수
- 예산 30억 원 규모
- 3개 기관 공동 참여

**중간 길이 개조식:**
- Saltlux, 연세대, KAIST와 공동으로 AI Professor System 개발 추진 중
- regional innovation 사업을 통한 지역 AI 혁신 생태계 구축이 핵심 목표

**긴 개조식:**
- 제주 지역의 도로 인프라 노후화에 대응하기 위해 AI 기반 도로 표면 점검 및 자율 보수 시스템을 서울대, KAIST, RovoRoad와 공동 개발하는 regional innovation 연계 프로젝트
- 우즈베키스탄 타슈켄트에 AI 교육센터를 설립하여 Microsoft Azure 인증 과정 포함 16주 커리큘럼을 운영하고, IT 전공 학생 20명을 대상으로 첫 코호트 런칭 예정

## 적용 지침

1. **모든 문장의 종결어 점검.** 초안 작성 후, 각 문장 끝이 명사/명사구로 끝나는지 확인. "~다", "~함", "~임", "~됨"이 보이면 명사형으로 변환.

2. **자연스러운 명사형 선택.** 기계적으로 종결어만 떼는 것이 아니라, 문장 전체의 흐름에서 자연스러운 명사 종결 탐색.

3. **불릿/번호 구조와 결합.** 개조식은 계층적 구조(1→가→(1)→(가) 또는 □→○→-)와 함께 사용할 때 가독성 극대화.

4. **조사 "~의", "~에 대한" 남용 회피.** 개조식에서도 간결한 수식이 중요. "AI 기반의 교육 시스템의 개발에 대한 추진" → "AI 기반 교육 시스템 개발 추진".

5. **주어 생략.** 맥락에서 주어가 명확하면 생략하여 밀도 향상.
