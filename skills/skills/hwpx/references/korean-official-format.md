# 한국 공문서 작성 규정

**근거 법령**: 「행정 효율과 협업 촉진에 관한 규정」(대통령령) 및 동 시행규칙 별표 서식. 구 「정부공문서규정」. 참고 편람: 행정안전부 「행정업무운영 편람」(최신판).

## 1. 공문서 종류

| 종류 | 정의 | 대표 예 |
|------|------|---------|
| **법규문서** | 법령 효력을 가진 문서 | 법률, 대통령령, 부령, 조례, 규칙 |
| **지시문서** | 상위 기관이 하위 기관에 업무 지시 | 훈령, 지시, 예규, 일일명령 |
| **공고문서** | 일정 사항을 공고하는 문서 | 고시, 공고 |
| **비치문서** | 행정기관에 비치하는 문서 | 비치대장, 비치카드 |
| **민원문서** | 민원인이 행정기관에 제출 또는 그 처리 결과 | 민원신청서, 민원회신 |
| **일반문서** | 위 다섯 종류 외 문서 | **기안문**, **보고서**, **사업계획서**, 회의록 |

스킬의 주 대상은 일반문서 — 특히 **기안문**(결재용 내부 문서)과 **사업계획서·보고서**.

## 2. 용지와 여백

- **용지**: A4 (210 × 297 mm), 세로
- **여백**:
  - 위: 30mm
  - 아래: 15mm
  - 왼쪽: 20mm
  - 오른쪽: 15mm

HWPX 페이지 설정(header.xml `<hp:pagePr>`)에서 HU 단위로 환산:
- 30mm = 8504 HU
- 20mm = 5668 HU
- 15mm = 4252 HU

## 3. 폰트 · 글자 크기 · 줄간격

**두 가지 관행이 병존한다**:

### 현행 기안문 (맑은 고딕 계열)
- **본문**: 맑은 고딕 11.5pt
- **제목**: 맑은 고딕 15pt 진하게
- **장평**: 100
- **자간**: 0
- **줄간격**: 160%

전자결재 시스템(온나라 2.0, K-Office, e-GIS)의 기본 템플릿이 이 스타일.

### 전통 보고서 (명조 계열)
- **본문**: 휴먼명조 15pt 또는 **함초롬바탕 15pt**
- **제목**: 휴먼명조 17~18pt
- **줄간격**: 160% 또는 200%

regional innovation·연구 보고서·사업 제안서에서 여전히 많이 쓰이며, Hancom Office 기본 폰트가 함초롬바탕이라 실무 편의성이 높다.

### 어느 것을 쓸지
- **공문·기안문** → 맑은 고딕 11.5pt (현행 표준)
- **보고서·사업계획서** → 휴먼명조/함초롬바탕 15pt (전통적 선호)
- 소속 기관 관행이 있으면 그것을 따를 것

## 4. 종결 방식 — 개조식

공문서 본문은 **개조식**(箇條式, 명사형 종결)으로 작성한다.

### 규칙
- 문장 끝을 "~함", "~임", "~할 것", "~해야 함"으로 마무리
- 서술어 "~합니다", "~입니다" 사용 금지 (대외시행 공문은 예외)
- 한 문장 한 조(條), 간결하게

### 예시

**잘못된 예 (서술체)**:
> 2026학년도 AI 교육센터 운영 계획을 수립합니다. 상반기에 교과목을 개편하고 하반기에는 실습 환경을 구축하겠습니다.

**올바른 예 (개조식)**:
> 2026학년도 AI 교육센터 운영 방향 확립함  
> 상반기 교과목 개편 및 교원 충원함  
> 하반기 실습 환경 구축 및 산학협력 확대함

작업용 스킬: `gaejosik` (vault에 별도 등록됨). 개조식 변환은 이 스킬에서 처리하고, HWPX 치환은 본 스킬에서 담당.

## 5. 4단계 번호 체계

본문 항목 번호는 **1. → 가. → 1) → 가)** 4단계.

```
1. 추진 목적
  가. 주요 내용
    1) 세부 항목
      가) 최하위 항목
```

들여쓰기: 각 단계별 2칸 들여쓰기 권장 (Hancom Office 기본값).

## 6. 기안문 구조 (10-part)

「시행규칙」 별표 서식 기준:

```
┌─────────────────────────────────────────┐
│   [기관 로고]  기  관  명                 │  ← 머리말
├─────────────────────────────────────────┤
│                                           │
│   수신  [수신자]   (수신처참조)            │
│   (경유)                                   │
│   제목  [제목]                             │
│                                           │
│   [본문 — 개조식, 4단계 번호]              │
│                                           │
│   붙임  1. [첨부 파일명] 1부.              │
│                                  끝.      │  ← "끝" 표시 필수
│                                           │
│                [발신명의]  [관인]          │  ← 대외시행 시
│                                           │
├─────────────────────────────────────────┤
│ 기안자 [성명]   검토자 [성명]   결재자 [성명]│
│ 시행 [문서번호] ([시행일자])                 │
│ 접수 [접수번호] ([접수일자])                 │
│ 우 [주소]  /  [홈페이지]                    │
│ 전화 [번호] / 팩스 [번호] / [이메일] / 공개구분│
└─────────────────────────────────────────┘
```

### 각 부분 상세

1. **머리말** (기관명 + 로고): 기안문 최상단. 전자결재 시스템이 자동 삽입.
2. **수신자**:
   - 내부결재: "내부결재"
   - 대외시행: 구체적 기관명 (예: "○○대학교 총장")
   - 다수 수신: "수신자 참조" + 붙임에 수신처 목록
3. **(경유)**: 경유 기관 있을 경우만 채움. 없으면 공란.
4. **제목**: 문서의 내용을 한 줄로 요약. "…계획(안)", "…보고", "…협조 요청" 식.
5. **본문**: 개조식, 4단계 번호.
6. **붙임**: 첨부 목록. "붙임  1. 파일명 1부."  마지막에 공백 2칸 후 "**끝.**" 필수.
7. **발신명의**: 대외시행 공문에만 등장. 오른쪽에 관인 위치 (전자결재가 자동 삽입).
8. **결재란**: 기안자/검토자/협조자/결재자. 전자결재 시스템이 자동 배치 — 스킬은 placeholder만 둠.
9. **시행·접수 정보**: 문서번호·일자. 시행은 기안 기관이, 접수는 수신 기관이 채움.
10. **주소·연락처**: 기관 주소, 홈페이지, 전화, 팩스, 이메일, 공개구분(공개/부분공개/비공개).

### "끝." 표시 규칙
- 본문 또는 붙임의 마지막 항목 뒤에 공백 두 칸 후 "끝." 으로 마무리
- 붙임이 있으면 붙임 마지막 줄 끝에, 없으면 본문 마지막 줄 끝에

## 7. 결재 방식

### 내부결재
- 수신자: "내부결재"
- 결재란: 기안자 → 검토자(팀장) → 협조자(타 부서) → 결재자(부서장/기관장)
- 전자결재 시스템 (온나라/K-Office)이 결재 화면 자체를 UI로 렌더 — 스킬은 placeholder만 표시

### 대외시행
- 내부결재 완료 후 수신 기관에 발송
- 발신명의: 기관장 직함 (예: "예시대학교 총장")
- 관인: 발신명의 오른쪽, e-결재 시스템이 전자이미지관인 자동 삽입 — **스킬은 관인 이미지 생성·삽입하지 않음**

## 8. 사업계획서 표준 구조 (9-section)

각 부처·사업단별 양식이 다르나 공통 뼈대:

```
I.   사업 개요        — 사업명, 기간, 총사업비, 주관·참여기관
II.  추진 배경 및 필요성 — 대내외 환경, 정책·기술 트렌드
III. 추진 목표 및 전략  — 최종목표, 연차별 목표, 추진전략
IV.  세부 추진 내용    — 세부과제 1/2/3 …
V.   추진 일정        — 연차별·월별 간트차트
VI.  소요 예산        — 인건비/사업비/관리비 내역
VII. 기대 효과        — 정량/정성 효과
VIII.성과 지표        — KPI, 측정 방법
IX.  붙임            — 참여진 이력, 예산 상세, 추진 체계도
```

## 9. 보고서 표준 구조

짧은 현황 보고는 4-section:
```
1. 추진 배경
2. 주요 내용
3. 향후 계획
4. 행정 사항
```

긴 보고서는 사업계획서 구조에 준함.

## 10. 페이지 번호

- 위치: **하단 중앙**
- 형식: "n / 총페이지수" 또는 숫자만
- 붙임은 별도 페이지 번호 부여 가능 (붙임-1, 붙임-2)

## 11. 정부 표준 서식 소스

공식 양식을 구하려면:

- **국가법령정보센터**: https://law.go.kr → "행정 효율과 협업 촉진에 관한 규정 시행규칙" → 별표/서식 (HWP 다운로드)
- **정부24**: https://gov.kr → 서식 민원
- **각 부처 홈페이지**: 사업 공고 첨부 HWP 양식
- **한국공공언어진흥원**: https://pikpl.or.kr → 2025 개정 공문서 작성법
- **행정안전부**: https://mois.go.kr → 행정업무운영 편람

본 스킬의 템플릿은 구조적 뼈대만 제공하므로, 최종 제출용으로는 소속 기관·사업단의 공식 HWP 양식을 내려받아 본 스킬로 anchor만 치환하는 방식을 권장한다.

## 12. 체크리스트

공문서 발송 전 확인:

- [ ] 용지 A4, 여백 30·15·20·15mm
- [ ] 폰트: 공문·기안문 = 맑은 고딕 11.5pt / 보고서 = 휴먼명조 15pt
- [ ] 줄간격 160%
- [ ] 본문 개조식 종결 ("~함", "~임")
- [ ] 4단계 번호: 1. → 가. → 1) → 가)
- [ ] 붙임 뒤 "끝." 표시
- [ ] 수신자 정확 (기관장 직함까지)
- [ ] 시행번호·시행일자 채움
- [ ] 공개구분 표시 (공개/부분공개/비공개)
- [ ] `./hwpx validate` 통과
- [ ] Hancom Office에서 1회 열어 서식 깨짐 없는지 확인 (자동화 불가 — 수동)
